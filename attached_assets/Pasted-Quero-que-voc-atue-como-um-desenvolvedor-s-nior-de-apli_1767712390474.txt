Quero que você atue como um desenvolvedor sênior de aplicativos (frontend + backend + arquitetura) e crie um aplicativo completo chamado **"Salva Plantão"**, voltado para médicos plantonistas.

O app deve ter:
- Área do usuário (médicos)
- Área administrativa exclusiva para o criador: **Dr. Eudes Rodrigues**
- Backend seguro com autenticação, banco de dados estruturado e controle de permissões
- Código completo, funcional e pronto para rodar em ambiente como Replit.

Plataforma alvo: Android e iOS. Sugira um stack adequado (ex: Flutter ou React Native + Firebase/Supabase/Node), mas entregue o projeto completo e funcional.

======================================================
====================   CONTEXTO   ====================
======================================================

O **Salva Plantão** é um app de apoio à rotina médica, com:

- Organização de plantões, agenda e controle financeiro.
- Prescrições e checklists de condutas.
- Área de estudo, anotações e resumos com escrita digitada e por caneta digital.
- Interconsulta com IA usando prompt definido pelo administrador.
- Chat entre usuários e chat direto com o administrador.
- Biblioteca de categorias com conteúdos e vídeos.
- Ferramentas extras úteis no plantão (calculadora flutuante, checklist de passagem, etc.)

Na tela inicial deve aparecer discretamente:
> **Criado por: Dr. Eudes Rodrigues**

======================================================
========= AUTENTICAÇÃO, ONBOARDING E TERMOS ==========
======================================================

Fluxo inicial obrigatório:
1. Cadastro/Login
2. Usuário deve ler e aceitar os **Termos de Uso**
   - Sem aceitar → não acessa o app

Gerar um texto-base de termos contendo:

- O app é ferramenta de apoio e organização, não substitui conduta médica formal.
- O chat entre usuários e a interconsulta com IA são apenas orientativos e educacionais.
- O administrador e o app não se responsabilizam por decisões clínicas tomadas pelo usuário.
- Proibido inserir dados sensíveis e identificáveis de pacientes.
- O usuário é totalmente responsável por seus atos profissionais.
- O administrador pode:
  - Bloquear usuários
  - Restringir chat
  - Encerrar acesso por mau uso

Os termos devem ser editáveis via painel do administrador.

======================================================
=========== INTERCONSULTA COM IA (MÓDULO IA) =========
======================================================

Aba exclusiva com:

1) Usuário pode inserir própria API Key de IA
2) ADMIN define o **prompt-base global**
3) Fluxo ao enviar caso clínico:
   - IA retorna:
     - 5 principais hipóteses diagnósticas numeradas
     - Diferenças entre elas (anamnese / exame / evolução)
   - Usuário escolhe uma hipótese
   - IA responde com:
     - Conduta completa
     - Exames necessários
     - Prescrição com doses
     - Dicas práticas

Aviso fixo na tela:
“Ferramenta educacional — decisão clínica é do médico usuário.”
