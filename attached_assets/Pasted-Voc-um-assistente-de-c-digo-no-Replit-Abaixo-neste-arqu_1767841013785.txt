### 1. ESTRUTURA DE DADOS / MODELO

Crie (ou atualize) a estrutura de dados de patologias seguindo um modelo semelhante a este (ajuste para o framework/linguagem já usado no projeto):

- Tabela/coleção `pathologies` com campos:
  - `id` (chave primária)
  - `name` (nome da patologia)
  - `age_group` (string: "adulto" ou "pediatrico")
  - `source_group` (ex.: "adulto_base", "pediatrico_base", "outras_prevalentes", "plantao_urgencia" – apenas para referência)
  - `clinical_category` (string: ex.: "Cardiologia", "Pneumologia", "Gastro/Hepato", "Nefro/Uro", "Endócrino/Metabólico", "Infectologia", "Reumatologia/Ortopedia", "Dermatologia", "Neurologia", "Psiquiatria", "Gineco/Obstetrícia", "Pediatria Geral", "Trauma/Urgência", "Toxicológico", "Outros")
  - `is_active` (boolean)

Use as listas fornecidas para:
- Inserir novas patologias
- Não duplicar nomes dentro do mesmo `age_group`
- Preencher o campo `clinical_category` de forma coerente com o nome (pode ser por mapeamento manual ou lógica simples baseada em palavras-chave, de forma robusta, sem travar o app).

### 2. ORGANIZAÇÃO POR ABA (ADULTO x PEDIATRIA)

O aplicativo já possui pelo menos duas abas: uma para ADULTO e outra para PEDIATRIA.

- Na aba **Adulto**:
  - Mostrar todas as patologias com `age_group = "adulto"`, organizadas por `clinical_category`.
- Na aba **Pediatria**:
  - Mostrar todas as patologias com `age_group = "pediatrico"`, organizadas por `clinical_category`.

Dentro de cada aba:
- Agrupar por `clinical_category`, por exemplo:
  - Cardíacas
  - Respiratórias
  - Gastro/Hepato
  - Nefro/Uro
  - Endócrino/Metabólico
  - Infecto
  - Reumato/Ortopedia
  - Trauma/Urgência
  - Pediatria Geral
  - etc.

A interface deve mostrar:
- Cabeçalho da categoria
- Lista de patologias daquela categoria
- Ações possíveis (ex.: abrir detalhes, vincular prescrições, duplicar entre abas)

### 3. BOTÃO DE "DUPLICAR PARA PEDIATRIA" / "DUPLICAR PARA ADULTO"

Em cada item de patologia, adicione:

- Na aba Adulto:
  - Um botão: **"Duplicar para Pediatria"**
- Na aba Pediatria:
  - Um botão: **"Duplicar para Adulto"**

Comportamento do botão:
1. Quando o usuário clicar em "Duplicar para Pediatria" em uma patologia adulta:
   - Verifique se já existe uma patologia com o MESMO `name` e `age_group = "pediatrico"`.
   - Se NÃO existir:
     - Crie um novo registro em `pathologies` com:
       - `name` igual
       - `age_group = "pediatrico"`
       - `clinical_category` adequado (pode herdar o mesmo ou ajustar se for necessário)
       - `source_group` marcado como algo como `"duplicado_de_adulto"`
   - Se já existir, apenas notificar visualmente que já existe na pediatria (sem criar duplicata).

2. Quando clicar em "Duplicar para Adulto" em uma patologia pediátrica:
   - Mesmo comportamento ao contrário:
     - Se não existir em adulto, criar.
     - Se já existir, não duplicar.

Importante:
- Esse botão NÃO apaga nada.
- Ele apenas facilita preencher a outra aba com a mesma patologia para poder configurar prescrições específicas.
